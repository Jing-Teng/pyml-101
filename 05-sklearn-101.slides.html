非常良善的 Scikit-Learn 設計¶

  • 一致性（Consistency）
  • 檢查性（Inspection）
  • 不自行創建類別（Nonproliferation of classes）
  • 模組化（Composition）
  • 提供合理的預設參數（Sensible defaults）

一致性（Consistency）¶

所有 sklearn 物件具有相同而單純的介面：

  • Estimators
    • Transformers.fit_transform()
    • Predictors.fit()
    • Predictors.predict()

檢查性（Inspection）¶

所有 Estimators 的參數都可以透過屬性 .ATTR 的方式檢視

不自行創建類別（Nonproliferation of classes）¶

資料格式都以 ndarray 儲存、參數都是 str、int 或 float

模組化（Composition）¶

多個 Predictors 或 Transformers 可以透過 Pipeline 與 Meta-Estimator 組裝起來

提供合理的預設參數（Sensible defaults）¶

在初始化 Predictor 的時候不需傳入參數即可創建基礎模型

根據監督式學習（Supervised Learning）的定義：¶

使用標籤資料（labelled data）訓練模型，再對無標籤資料（unlabelled data）預測

所以我們可以使用 train.csv 做訓練，然後對 test.csv 做預測對吧？

是，也不是¶

  • （是）Kaggle 的 Getting Started 競賽允許一天上傳多次對答案
  • （不是）生活中的專案應用「對答案」這件事情可能需要花費時間（例如：A/B Test 的實驗設計）

因此通常會將 train.csv 再切分成為訓練資料與驗證資料兩個部分¶

  • 訓練（train）是標籤資料（labelled data）
  • 驗證（validation）是偽裝成無標籤資料（unlabelled data）的標籤資料（labelled data）

import math

def my_train_test_split(X, test_size, random_state):
    X_indices = np.array(X.index)
    r = np.random.RandomState(random_state)
    r.shuffle(X_indices)
    test_set_size = math.ceil(X_indices.size * test_size)
    validation_indices = X_indices[:test_set_size]
    train_indices = X_indices[test_set_size:]
    return X.iloc[train_indices, :], X.iloc[validation_indices, :]

def my_train_test_split(X, y, test_size, random_state):
    X_indices = np.arange(y.size)
    r = np.random.RandomState(random_state)
    r.shuffle(X_indices)
    test_set_size = math.ceil(X_indices.size * test_size)
    validation_indices = X_indices[:test_set_size]
    train_indices = X_indices[test_set_size:]
    return X[train_indices, :], X[validation_indices, :], y[train_indices], y[validation_indices]

由於隨機排序的關係，訓練與驗證資料的分割會影響模型的評估指標¶

In [29]:
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

boston = load_boston()
X = boston["data"]
y = boston["target"]
mses = []
n_iters = range(10)
for _ in n_iters:
    X_train, X_validation, y_train, y_validation = train_test_split(X, y, test_size=0.33) # Calling without random_state
    lr = LinearRegression()
    lr.fit(X_train, y_train)
    y_pred = lr.predict(X_validation)
    mses.append(mean_squared_error(y_validation, y_pred))

在 K=5 的設定下，我們將會把資料依據 80:20 的比例分割為訓練與驗證，並迭代五次¶

$$CV_{MSE} = \frac{1}{K} \sum_{i=1}^{K}MSE_i$$
In [35]:
boston = load_boston()
X = boston["data"]
y = boston["target"]
kf = KFold(n_splits=5)
mses = []
for train_indices, validation_indices in kf.split(X):
    X_train, X_validation, y_train, y_validation = X[train_indices, :], X[validation_indices, :], y[train_indices], y[validation_indices]
    lr = LinearRegression()
    lr.fit(X_train, y_train)
    y_pred = lr.predict(X_validation)
    mses.append(mean_squared_error(y_validation, y_pred))
print(mses)
print(sum(mses) / len(mses))

使用 KFold() 函數之後得到的只是分割的索引值依據，$CV_{MSE}$ 得自己算¶

可以再進階使用 sklearn.model_selection 的 cross_val_score()

In [36]:
from sklearn.model_selection import cross_val_score

boston = load_boston()
X = boston["data"]
y = boston["target"]
lr = LinearRegression()
kf = KFold(n_splits=5) 
neg_mses = cross_val_score(lr, X, y, cv=kf, scoring='neg_mean_squared_error')
mses = [-i for i in neg_mses]
print(mses)
print(sum(mses) / len(mses))
